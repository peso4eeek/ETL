# ETL Pipeline: PostgreSQL → MongoDB (Background Replication)

Асинхронный отказоустойчивый ETL-воркер для фоновой репликации данных из реляционной СУБД (PostgreSQL) в документоориентированную NoSQL (MongoDB). 

## Архитектура системы
Система разворачивается в изолированной среде через `docker-compose` и состоит из трех узлов:
1. **Source DB (PostgreSQL):** Источник данных. При первом запуске автоматически инициализируется скриптом генерации высоконагруженных данных (~500 000 строк).
2. **Destination DB (MongoDB):** Приемник данных. Хранит иерархические документы.
3. **ETL Worker:** Фоновый сервис (BackgroundService), управляющий процессом синхронизации.

## ETL worker реализует следующие архитектурные решения:

* **Инкрементальная загрузка:** Воркер запоминает время последней синхронизации и запрашивает через легковесный `Dapper` только те строки, где `updated_at >= LastSync` или `deleted_at >= LastSync`.
* **Продвинутая трансформация (Transform):** Проблема декартова произведения при SQL `JOIN` (Many-to-Many) решается через группировку в памяти C# (`GroupBy`, `DistinctBy`). На выходе формируется единый JSON-документ (Группа), содержащий внутри себя массивы Студентов и Преподавателей.
* **Идемпотентность:** Повторное выполнение скрипта на одних и тех же данных не приводит к дублированию. В MongoDB используется операция `ReplaceOne` с флагом `IsUpsert = true`.
* **Обработка удалений :** Реализована полноценная поддержка `deleted_at`. Если сущность удаляется в источнике, воркер каскадно исключает её из документа MongoDB или полностью удаляет документ (`DeleteOne`), если удален корень агрегата.
* **Отказоустойчивость:** В docker-compose настроен `healthhcheck` для postgres. Перед выполнением скрипта worker проверяет соединение с базой данных.

##  Быстрый старт

**1. Настройка окружения:**
Убедитесь, что в корне проекта (или рядом с docker-compose) лежит файл `.env` с переменными.
```env
SYNC_INTERVAL_MINUTES=1
POSTGRES_USER=postgres
POSTGRES_PASSWORD=password
POSTGRES_DB=etldb
MONGO_INITDB_ROOT_USERNAME=mongouser
MONGO_INITDB_ROOT_PASSWORD=mongopassword
```
2. Запуск инфраструктуры:
```Bash
docker-compose up -d
```
> **Примечание:** При первом запуске PostgreSQL требуется около 10-15 секунд на генерацию 500 000 строк и построение индексов. ETL-воркер автоматически дождется окончания этого процесса.

Убедиться, что скрипт отработал можно следующим образом:
1. Посмотреть логи etl-workera, убедившись в том, что синхронизация прошла успешно:
```Bash
docker-compose logs -f etl-worker
```
Вы должны увидеть логи об успешной обработке и репликации данных.

2. Зайдите внутрь контейнера Mongo и запустите интерактивную оболочку:

```Bash
docker exec -it <имя_контейнера_mongo> mongosh -u mongouser -p mongopassword
```
Выполните следующие команды для проверки структуры:
```JavaScript
// 1. Переключаемся на базу
use university;

// 2. Проверяем количество перенесенных агрегатов (Должно быть 10000 групп)
db.groups.countDocuments();

// 3. Смотрим структуру агрегатированного документа
db.groups.findOne();
```
